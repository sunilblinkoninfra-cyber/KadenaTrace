# KadenaTrace Demo Script

## Intro: the problem

**What to say**

"Crypto fraud moves fast. Victims usually have a wallet address or a suspicious transaction, but not a clear picture of where funds went, how quickly they moved, or whether the investigation can be independently verified. KadenaTrace turns that raw input into a forensic graph, explainable risk findings, and a Kadena-verifiable public record."

## System overview

**What to click**

1. Open the app at `http://localhost:3000`.
2. Point out the search form, the public case pages, and the attestations dashboard.

**What to say**

"This system is split into three layers. Off-chain workers crawl transaction flows and score suspicious behavior. The API serves graph data, risk analysis, and deterministic trace hashes. Kadena Pact stores only integrity commitments, so the public can verify a case without putting private investigation details on-chain."

## Live demo steps

### Step 1: Generate the trace

**What to click**

1. Set `Chain` to `Ethereum`.
2. Set `Seed type` to `Compromised wallet`.
3. Paste `0x1111111111111111111111111111111111111111`.
4. Click `Generate Trace`.

**What to say**

"I’m starting from a single compromised wallet. The user only provides the seed. The system does the graph expansion, risk scoring, and trace hashing automatically."

### Step 2: Walk the graph

**What to click**

1. Show the risk legend above the graph.
2. Change the risk filter from `All` to `Medium`.
3. Click a suspicious path in the side panel.
4. Use `+`, `-`, and `Reset view`.

**What to say**

"The graph is interactive and explainable. Red and orange paths show higher risk movement. Purple highlights show bridge interaction. Clicking a suspicious path focuses the exact laundering route instead of forcing an investigator to inspect every edge manually."

### Step 3: Explain the risk

**What to click**

1. Scroll to the `Findings` section.
2. Point to the risk verdict banner.
3. Open a node detail panel if available and show the score breakdown.

**What to say**

"Every risk call is explainable. We’re not just saying this wallet is bad. We show the signals that triggered the score: fan-out bursts, rapid hops, bridge usage, mixer interaction, and sink consolidation. That makes the output reviewable by investigators, exchanges, or judges."

### Step 4: Show verification

**What to click**

1. Point to the verification badge on the trace page.
2. Point to the trace hash.
3. Open the terminal and run `npm run verify-trace -- 28f0858faf38d464688842ee0390aa908e21ccf7e2a8d443a205e3d527b9c966`.

**What to say**

"This trace is deterministic. The same graph edges always produce the same SHA-256 trace hash. That means the investigation can be independently recomputed and verified outside the UI. The CLI checks the stored trace against the recomputed commitment."

### Step 5: Publish the case

**What to click**

1. In `Publish Investigation`, click `Create Public Case`.
2. Open the generated public case link.
3. If a wallet is connected, click `Sign & Relay on Kadena`.

**What to say**

"Now we turn the private investigation into a public audit artifact. The case page shows the frozen graph, findings, and attestations. When we anchor it on Kadena, we commit hashes only, not raw case data, so the chain proves integrity without leaking sensitive context."

### Step 6: Show public outputs

**What to click**

1. Open `/case/shadow-router-laundering-pattern`.
2. Open `/attestations`.
3. Mention `examples/real-scam-case.json` in the repo.

**What to say**

"Judges can inspect the public case page, the aggregate attestation dashboard, and a realistic sample case file in the repo. That makes the project easy to review both as a product and as an engineering system."
